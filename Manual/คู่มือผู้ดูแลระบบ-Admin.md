## 1. ภาพรวมสิทธิ์ Admin

Admin มีสิทธิ์ทุกอย่างที่ Owner และ Member มี **บวกเพิ่ม** ดังนี้:

| สิทธิ์พิเศษ | รายละเอียด |
|-------------|-----------|
| **จัดการผู้ใช้** | สร้าง/แก้ไข/ลบ user, เปลี่ยน Role, Reset Password |
| **จัดการกลุ่ม** | สร้าง/แก้ไข/ลบ Group, เพิ่มสมาชิก, กำหนดโครงการ |
| **เข้าถึง Configuration** | ตั้งค่าระบบ |
| **ข้ามสิทธิ์โครงการ** | แก้ไข/ลบโครงการทุกตัวได้ แม้ไม่ได้เป็นสมาชิก |
| **ข้ามสิทธิ์งาน** | แก้ไข/ลบ Task ทุกตัวได้ข้ามโครงการ |

> **หมายเหตุ:** สำหรับการใช้งานทั่วไป (Dashboard, Projects, My Tasks, Calendar, Timeline) ให้อ่าน **คู่มือผู้ใช้งาน Owner-Member** ควบคู่กัน เอกสารนี้จะอธิบายเฉพาะส่วนที่ Admin มีเพิ่มเติม

## 2. การเข้าสู่ระบบ

เหมือนกับผู้ใช้ทั่วไป:

1. เปิด Browser ไปที่ URL ระบบ
2. กรอก Email และ Password
3. คลิก **Sign In**

> Admin จะเห็นเมนูเพิ่มเติม 2 รายการใน Sidebar: **Groups** และ **Configuration**

## 5. User Management — จัดการผู้ใช้งาน

### 5.1 เข้าสู่หน้าจัดการผู้ใช้

1. คลิก **Configuration** ที่ Sidebar
2. คลิก **User Management**
3. ระบบแสดงหน้า Users พร้อมตารางข้อมูล

### 5.2 ตาราง Role Permissions (ด้านบน)

ตารางอ้างอิงสิทธิ์แต่ละ Role:

| Role | สิทธิ์ |
|------|--------|
| **ADMIN** | เข้าถึง Configuration/User management, แก้ไข/ลบ project ทุกตัวได้ (ไม่ต้องเป็นสมาชิก), จัดการ tasks ข้ามโครงการได้ |
| **OWNER** | เจ้าของโครงการ, สิทธิ์เทียบเท่า ADMIN ในเรื่อง project/task, เพิ่ม/ลบสมาชิกในโครงการตัวเองได้, แต่เข้า Configuration ไม่ได้ |
| **MEMBER** | สมาชิกทั่วไป, เห็น/แก้ไขเฉพาะ project ที่ตัวเองเป็นสมาชิก, ทำ task ที่ assign ให้ได้ |

### 5.3 ตารางผู้ใช้งาน

แสดงรายชื่อผู้ใช้ทั้งหมดในระบบ:

| คอลัมน์ | รายละเอียด |
|---------|-----------|
| Name | ชื่อผู้ใช้ |
| Email | อีเมล |
| Role | บทบาท (ADMIN / OWNER / MEMBER) |
| Actions | ปุ่ม Edit และ Reset Password |

### 5.4 แก้ไข Role ผู้ใช้

1. คลิกปุ่ม **Edit** (ไอคอนดินสอ) ที่แถวของผู้ใช้
2. เลือก Role ใหม่จาก dropdown: **ADMIN** / **MEMBER**
3. คลิก **Save**

> **คำเตือน:** การเปลี่ยน Role มีผลทันที ผู้ใช้จะเห็น/ไม่เห็นเมนูตาม Role ใหม่เมื่อ refresh หน้า

### 5.5 Reset Password ผู้ใช้

ใช้เมื่อผู้ใช้ลืมรหัสผ่านหรือต้องการตั้งรหัสผ่านใหม่:

1. คลิกปุ่ม **Reset Password** (ไอคอนกุญแจ) ที่แถวของผู้ใช้
2. ระบบเปิด Modal ให้กรอกรหัสผ่านใหม่
3. พิมพ์รหัสผ่านใหม่ (อย่างน้อย 6 ตัวอักษร)
4. คลิก **Reset** หรือกด Enter

> **หมายเหตุ:** แจ้งผู้ใช้ให้ทราบรหัสผ่านใหม่ทางช่องทางที่ปลอดภัย

## 6. Groups — จัดการกลุ่ม

คลิกเมนู **Groups** ที่ Sidebar

### 6.1 หน้ารวม Groups

แสดงรายการกลุ่มทั้งหมด:
- ชื่อกลุ่ม, คำอธิบาย, ประเภท
- จำนวนสมาชิก, จำนวนโครงการ
- สีประจำกลุ่ม

### 6.2 สร้าง Group ใหม่

1. คลิกปุ่ม **+ New Group**
2. กรอกข้อมูล:

| ช่อง | บังคับ | รายละเอียด |
|------|--------|-----------|
| Group Name | ใช่ | ชื่อกลุ่ม |
| Description | ไม่ | คำอธิบายกลุ่ม |
| Type | ไม่ | ประเภทกลุ่ม (เลือกจาก dropdown) |
| Color | ไม่ | สีประจำกลุ่ม |

3. คลิก **Create**

### 6.3 แก้ไข Group

1. คลิกชื่อกลุ่มเพื่อเปิดรายละเอียด
2. คลิกปุ่ม **Edit**
3. แก้ไขข้อมูลที่ต้องการ
4. คลิก **Save**

### 6.4 ลบ Group

1. คลิกจุด 3 จุด (⋮) หรือปุ่มลบที่กลุ่มนั้น
2. ยืนยันการลบ

> **คำเตือน:** การลบกลุ่มจะยกเลิกการเชื่อมโยงสมาชิกและโครงการออกจากกลุ่ม

### 6.5 เพิ่มสมาชิกเข้ากลุ่ม

1. เปิดรายละเอียดกลุ่ม
2. ที่ส่วน **Members** → เลือกผู้ใช้จาก dropdown
3. คลิก **Add**

### 6.6 ลบสมาชิกออกจากกลุ่ม

1. ที่ส่วน Members → คลิกไอคอนถังขยะข้างชื่อสมาชิก
2. ยืนยันการลบ

### 6.7 กำหนดโครงการให้กลุ่ม

1. ที่ส่วน **Projects** → เลือกโครงการจาก dropdown
2. คลิก **Add**

### 6.8 ลบโครงการออกจากกลุ่ม

1. ที่ส่วน Projects → คลิกไอคอนถังขยะข้างชื่อโครงการ
2. ยืนยันการลบ

## 7. สิทธิ์พิเศษของ Admin ในการจัดการโครงการ

Admin สามารถทำสิ่งต่อไปนี้กับ **โครงการทุกตัว** แม้ไม่ได้เป็นสมาชิก:

| การดำเนินการ | Member | Owner | Admin |
|-------------|:------:|:-----:|:-----:|
| ดูโครงการที่เป็นสมาชิก | ใช่ | ใช่ | ใช่ |
| ดูโครงการทุกตัว | ไม่ | ไม่ | **ใช่** |
| สร้างโครงการ | ใช่ | ใช่ | ใช่ |
| แก้ไขโครงการตัวเอง | ใช่ | ใช่ | ใช่ |
| แก้ไขโครงการที่ไม่ได้เป็นสมาชิก | ไม่ | ไม่ | **ใช่** |
| ลบโครงการตัวเอง | ใช่ | ใช่ | ใช่ |
| ลบโครงการที่ไม่ได้เป็นสมาชิก | ไม่ | ไม่ | **ใช่** |
| เปลี่ยนสถานะโครงการทุกตัว | ไม่ | ไม่ | **ใช่** |

### วิธีใช้

ไม่มีขั้นตอนพิเศษ — Admin จะเห็นโครงการทุกตัวในหน้า Projects และสามารถ:
- คลิก Edit / Delete ได้เลย
- ลากเปลี่ยนสถานะใน Board View ได้
- เข้าดูรายละเอียดและจัดการ Task ภายในได้

## 8. สิทธิ์พิเศษของ Admin ในการจัดการงาน

Admin สามารถทำสิ่งต่อไปนี้กับ **Task ทุกตัว** ข้ามโครงการ:

| การดำเนินการ | Member | Owner | Admin |
|-------------|:------:|:-----:|:-----:|
| ดู Task ที่ assign ให้ | ใช่ | ใช่ | ใช่ |
| ดู Task ทุกตัว | ไม่ | ในโครงการตัวเอง | **ทุกโครงการ** |
| สร้าง Task | ในโครงการที่เป็นสมาชิก | ใช่ | **ทุกโครงการ** |
| แก้ไข Task | ที่ assign ให้ | ในโครงการตัวเอง | **ทุกตัว** |
| ลบ Task | ที่สร้างเอง | ในโครงการตัวเอง | **ทุกตัว** |
| เปลี่ยนสถานะ/Priority | ที่ assign ให้ | ในโครงการตัวเอง | **ทุกตัว** |
| เพิ่ม/เปลี่ยน Assignees | ไม่ | ในโครงการตัวเอง | **ทุกตัว** |

## 10. คำแนะนำสำหรับ Admin

### 10.1 การจัดการผู้ใช้

- **สร้าง User ใหม่:** แจ้ง Email และ Password เริ่มต้นให้ผู้ใช้ ให้เปลี่ยนรหัสผ่านทันทีหลัง login ครั้งแรก
- **กำหนด Role อย่างเหมาะสม:**
  - **ADMIN** — เฉพาะผู้ดูแลระบบที่ต้องจัดการ user/group
  - **OWNER** — หัวหน้าโครงการที่ต้องจัดการสมาชิกและ task ในโครงการ
  - **MEMBER** — สมาชิกทั่วไปที่ทำงานตาม task ที่ assign
- **Reset Password:** ใช้เมื่อผู้ใช้ลืมรหัสผ่าน กำหนดรหัสชั่วคราวแล้วแจ้งผู้ใช้

### 10.2 การจัดการกลุ่ม

- ใช้ **Groups** เพื่อจัดกลุ่มทีมตามแผนก/ฟังก์ชัน
- กำหนดโครงการให้กลุ่มเพื่อให้เห็นภาพรวมว่าแต่ละทีมรับผิดชอบอะไร
- อัปเดตสมาชิกในกลุ่มเมื่อมีการเปลี่ยนแปลงทีม

### 10.3 การดูแลโครงการภาพรวม

- ใช้หน้า **Timeline** เพื่อดูแผนงานรายปีทุกโครงการ
- ตรวจสอบโครงการที่มีสถานะ **Delay** เป็นประจำ
- ดู **Progress %** เพื่อติดตามความคืบหน้า
- Export ข้อมูลเป็น Excel/PDF เพื่อส่งรายงานฝ่ายบริหาร

### 10.4 หลักปฏิบัติด้านความปลอดภัย

- **ไม่ควรมี Admin หลายคนเกินไป** — จำกัดเฉพาะผู้ที่จำเป็น
- **ตรวจสอบ User list เป็นประจำ** — ลบ user ที่ไม่ได้ใช้แล้ว
- **ใช้รหัสผ่านที่แข็งแรง** — แนะนำให้ผู้ใช้ตั้งรหัสผ่าน 8 ตัวอักษรขึ้นไป
- **เปลี่ยนรหัสผ่าน Admin เป็นประจำ**
